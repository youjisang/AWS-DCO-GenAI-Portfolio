# AWS DCO 인턴십 직무 이해를 위한 생성형 AI 활용 포트폴리오

## 프로젝트 소개

이 프로젝트는 공개된 DCO 직무 정보와 교육용 샘플 데이터를 바탕으로
DCO 직무의 기본 용어, 교육용 로그 분석, 장애 상황 문서화와 인수인계 흐름을 학습한 결과물입니다.

생성형 AI를 단순히 답을 생성하는 도구로 사용하지 않고,
용어 정리, 문서 구조화, 로그 분석, Python 코드 작성, 보고서 초안 작성과 결과 검증을 돕는 도구로 활용했습니다.

이 프로젝트는 실제 AWS 내부 시스템, 실제 운영 로그, 실제 Ticket 또는 실제 데이터센터 장비를 대상으로 수행한 프로젝트가 아닙니다.

## 생성형 AI 활용 원칙

* 생성형 AI의 답변은 최종 정답이 아니라 검토가 필요한 초안으로 사용했습니다.
* AI가 작성한 장비명, 시간, Ticket ID와 사건 내용을 원본 로그와 비교했습니다.
* 로그에서 직접 확인되는 사실과 가능한 추정을 구분했습니다.
* 로그만으로 확인할 수 없는 Root Cause는 확정하지 않았습니다.
* Ticket 우선순위는 AI가 대신 결정하지 않고, 학생이 근거를 비교하여 직접 판단했습니다.
* 실제 계정정보, 실제 장비정보, 실제 Ticket과 실제 AWS 내부정보를 AI에 입력하지 않았습니다.
* 모든 로그, 장비명, Ticket과 IP 주소는 교육용 샘플 데이터로 구성했습니다.

## 배운 점

* 생성형 AI를 활용하면 로그 요약, 코드 작성과 문서 초안 작성 시간을 줄일 수 있습니다.
* AI가 빠르게 결과를 생성하더라도 원본 자료와 비교하는 검증 과정이 필요합니다.
* 로그 분석에서는 확인된 사실, 가능한 추정과 미확인 정보를 구분해야 합니다.
* 장애가 복구되었다는 사실만으로 정확한 Root ause가 확인된 것은 아닙니다.
* Severity는 사건의 심각도이고, Priority는 무엇을 먼저 확인할 것인지 정한 순서입니다.
* Ticket 우선순위는 서비스 영향, 반복·지속 여부, 영향 범위, 복구 여부와 현재 상태를 함께 고려해야 합니다.
* Incident Report는 사건의 흐름과 조치 결과를 다른 사람이 이해할 수 있도록 정리해야 합니다.
* Handover Memo는 교대 전에 진행 중인 상황과 다음 확인 사항을 다음 담당자에게 전달하기 위해 작성합니다.
* 기술 결과물뿐 아니라 AI를 어떤 목적으로 사용하고, 무엇을 직접 검증하고 수정했는지를 기록하는 것이 중요합니다.

## 프로젝트 범위와 제한

이 저장소는 DCO 인턴십 직무를 이해하기 위한 학습형 포트폴리오입니다.

* 실제 AWS DCO 업무 수행 능력을 증명하는 자료가 아닙니다.
* 실제 데이터센터 운영 절차나 내부 SOP를 포함하지 않습니다.
* 실제 장애 대응, 장비 교체 또는 운영 자동화를 수행하지 않습니다.
* 공개 직무 맥락과 교육용 샘플 데이터를 이용한 학습 결과만 포함합니다.
